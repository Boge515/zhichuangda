# 个人中心功能实现步骤文档

## 项目背景
在智创答AI应用平台的前端项目中，需要在用户登录后，点击右上角的用户名跳转到个人中心页面，用于编辑个人信息。

## 实现目标
1. 创建个人中心页面组件
2. 添加个人中心路由配置
3. 修改全局头部组件，使登录用户的用户名可点击跳转
4. 优化页面布局样式

## 详细实现步骤

### 第一步：分析项目结构和现有路由

#### 1.1 查看项目结构
- 项目使用 Vue 3 + TypeScript + Arco Design Vue 组件库
- 路由配置在 `src/router/routes.ts` 文件中
- 用户相关页面位于 `src/views/user/` 目录下
- 全局头部组件在 `src/components/GlobalHeader.vue`

#### 1.2 检查现有路由
查看 `routes.ts` 文件，发现已有用户相关路由：
- `/user/login` - 用户登录
- `/user/register` - 用户注册
- 用户路由使用 `UserLayout` 布局组件

#### 1.3 检查用户存储和API
- Pinia，位于 `src/store/userStore.ts`
- 用户API接口在 `src/api/userController.ts` 中
- 有 `updateMyUserUsingPost` 接口可用于更新用户信息

### 第二步：创建个人中心页面组件

#### 2.1 创建文件
在 `src/views/user/` 目录下创建 `UserCenterPage.vue` 文件

#### 2.2 页面结构设计

##### 脚本部分
```typescript
<script setup lang="ts">
import { ref, reactive, onMounted } from "vue";
import { useLoginUserStore } from "@/store/userStore";
import { updateMyUserUsingPost } from "@/api/userController";
import message from "@arco-design/web-vue/es/message";
import { useRouter } from "vue-router";
import ACCESS_ENUM from "@/access/accessEnum";

// 初始化数据
const loginUserStore = useLoginUserStore();
const router = useRouter();
const loading = ref(false);

// 用户信息
const userInfo = ref({
  id: 0,
  userName: "",
  userAvatar: "",
  userProfile: "",
  userRole: "",
  createTime: "",
});

// 表单数据
const form = reactive({
  userName: "",
  userAvatar: "",
  userProfile: "",
});

// 初始化用户数据
const initUserData = () => {
  const user = loginUserStore.loginUser;
  userInfo.value = {
    id: user.id || 0,
    userName: user.userName || "",
    userAvatar: user.userAvatar || "",
    userProfile: user.userProfile || "",
    userRole: user.userRole || "",
    createTime: user.createTime || "",
  };
  
  form.userName = user.userName || "";
  form.userAvatar = user.userAvatar || "";
  form.userProfile = user.userProfile || "";
};

// 提交表单
const handleSubmit = async () => {
  loading.value = true;
  try {
    const res = await updateMyUserUsingPost({
      userName: form.userName,
      userAvatar: form.userAvatar,
      userProfile: form.userProfile,
    });
    
    if (res.data.code === 0) {
      message.success("个人信息更新成功");
      await loginUserStore.fetchLoginUser();
    } else {
      message.error("更新失败：" + res.data.message);
    }
  } catch (error) {
    message.error("更新失败，请稍后重试");
    console.error(error);
  } finally {
    loading.value = false;
  }
};

// 取消按钮
const handleCancel = () => {
  router.back();
};

// 组件挂载时初始化数据
onMounted(() => {
  initUserData();
});
</script>
```

## 常见问题及解决方案

### Q1: 路由跳转后页面不显示？
A: 检查路由配置是否正确，确保组件导入路径正确，路由路径没有冲突。

### Q2: 用户信息不显示？
A: 检查用户存储是否已初始化，确保登录状态正确，检查 API 调用是否成功。

### Q3: 样式不对齐？
A: 检查容器宽度设置，确保所有表单字段使用相同的宽度单位，使用浏览器开发者工具调试布局。

### Q4: 权限控制不生效？
A: 检查路由的 `meta.access` 配置，确保权限检查函数 `checkAccess` 正常工作。
